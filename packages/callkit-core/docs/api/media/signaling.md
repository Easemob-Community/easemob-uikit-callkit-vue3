# 信令交互

## 单聊信令流程

### 正常接通流程

```
主叫方                              被叫方
  │                                   │
  │  1. invite (msg)                  │
  │ ────────────────────────────────► │
  │                                   │
  │         2. alert (cmd)            │
  │ ◄──────────────────────────────── │
  │                                   │
  │  3. confirmRing (cmd)             │
  │ ────────────────────────────────► │
  │                                   │
  │         4. answerCall (cmd)       │
  │ ◄──────────────────────────────── │
  │  (result: 'accept')               │
  │                                   │
  │  5. confirmCallee (cmd)           │
  │ ────────────────────────────────► │
  │                                   │
  │  ┌─────────────────────────────┐  │
  │  │      双方进入 IN_CALL        │  │
  │  └─────────────────────────────┘  │
  │                                   │
  │  6. leaveCall (cmd)               │
  │ ────────────────────────────────► │
  │  (或反向)                         │
  │                                   │
```

### 拒绝流程

```
主叫方                              被叫方
  │                                   │
  │  1. invite (msg)                  │
  │ ────────────────────────────────► │
  │                                   │
  │         2. answerCall (cmd)       │
  │ ◄──────────────────────────────── │
  │  (result: 'refuse')               │
  │                                   │
  │  ┌─────────────────────────────┐  │
  │  │   主叫收到 callRefused 事件   │  │
  │  │   双方回到 IDLE              │  │
  │  └─────────────────────────────┘  │
```

### 忙线流程

当被叫方正处于通话中时，自动回复 busy：

```
主叫A                             被叫B (正在通话中)
  │                                   │
  │  1. invite (msg)                  │
  │ ────────────────────────────────► │
  │                                   │
  │         2. answerCall (cmd)       │
  │ ◄──────────────────────────────── │
  │  (result: 'busy')                 │
  │                                   │
  │  ┌─────────────────────────────┐  │
  │  │   主叫A收到 callBusy 事件     │  │
  │  │   主叫A回到 IDLE             │  │
  │  └─────────────────────────────┘  │
```

### 超时流程

主叫发出 invite 后，若在 `inviteTimeout`（默认 30s）内未收到 answerCall：

```
主叫方
  │
  │  1. invite (msg)
  │ ────────────────────────────────► 被叫方
  │
  │  ... 30s 内无响应 ...
  │
  │  ┌─────────────────────────────┐
  │  │   触发 callTimeout 事件       │
  │  │   触发 callEnded 事件         │
  │  │   回到 IDLE                  │
  │  └─────────────────────────────┘
```

## 群聊信令流程

### 发起与加入流程

```
主叫方                              被叫方1                            被叫方2
  │                                   │                                  │
  │  1. invite (msg, 群聊ext)         │                                  │
  │ ──────────────────────────────────┼────────────────────────────────► │
  │                                   │                                  │
  │         2. answerCall (cmd)       │                                  │
  │ ◄─────────────────────────────────│                                  │
  │  (result: 'accept')               │                                  │
  │                                   │                                  │
  │  ┌────────────────────────────────────────────────────────────────┐ │
  │  │   主叫收到 participantStateChanged (accepted)                   │ │
  │  │   所有参与者收到 groupCallInit 事件                             │ │
  │  └────────────────────────────────────────────────────────────────┘ │
  │                                   │                                  │
  │  3. confirmCallee (cmd)           │                                  │
  │ ─────────────────────────────────►│                                  │
  │                                   │                                  │
  │         4. answerCall (cmd)       │                                  │
  │ ◄─────────────────────────────────┼─────────────────────────────────│
  │  (result: 'accept')               │                                  │
```

### 成员离开流程

群聊中任意成员离开不影响其他成员继续通话：

```
成员A                              成员B                              成员C
  │                                   │                                  │
  │  1. leaveCall (cmd)               │                                  │
  │ ──────────────────────────────────┼────────────────────────────────► │
  │                                   │                                  │
  │  ┌────────────────────────────────────────────────────────────────┐ │
  │  │   所有成员收到 participantLeft 事件                             │ │
  │  │   成员A从参与者列表移除                                        │ │
  │  │   成员B、C继续通话                                             │ │
  │  └────────────────────────────────────────────────────────────────┘ │
```

## 消息格式

### invite (消息体)

```json
{
  "type": "cmd",
  "action": "invite",
  "from": "caller_user_id",
  "to": "callee_user_id",
  "ext": {
    "callId": "uuid",
    "callType": 0,
    "channel": "random_channel_name",
    "callerDevId": "device_id",
    "calleeDevId": "target_device_id",
    "timestamp": 1234567890
  }
}
```

### answerCall (命令消息)

```json
{
  "type": "cmd",
  "action": "answerCall",
  "from": "callee_user_id",
  "to": "caller_user_id",
  "ext": {
    "callId": "uuid",
    "result": "accept",
    "calleeDevId": "device_id"
  }
}
```

### leaveCall (命令消息)

```json
{
  "type": "cmd",
  "action": "leaveCall",
  "from": "user_id",
  "to": "target_user_id",
  "ext": {
    "callId": "uuid",
    "reason": "normal"
  }
}
```

## 超时与过期处理

### 邀请超时

- 配置项：`CallKitCoreConfig.inviteTimeout`（默认 30000ms）
- 触发时机：主叫发出 invite 后，在超时时间内未收到被叫的 answerCall
- 结果：自动触发 `callTimeout` + `callEnded` 事件，状态回到 IDLE

### 消息过期过滤

- 场景：用户重新登录后，可能收到离线期间的旧 invite
- 机制：收到 invite 时校验消息时间戳，若超过 `inviteTimeout + 10000ms` 则忽略
- 日志：`[邀请处理] 邀请消息已过期，忽略`

### CMD 消息过期过滤

- 场景：网络延迟导致 CMD 消息乱序到达
- 机制：收到 CMD 消息时校验，若消息时间早于当前通话的 invite 时间则忽略
- 防止：旧通话的 cancel/leave 影响新通话
