## 简介
本测试指南面向 EaseMob Chat CallKit Vue3 组件库，系统性阐述测试架构与流程，重点覆盖两种测试模式：
- 源码模式测试（test:source）：直接基于 lib 源码开发与运行，便于联调与快速迭代
- tgz 包测试（test:tgz）：通过打包产物 .tgz 安装为依赖进行验证，模拟真实发布态

同时，文档将解释测试环境配置差异、切换机制（switch-mode.mjs）、测试脚本工作原理，并给出单元测试、集成测试与端到端测试的最佳实践，以及测试覆盖率与报告生成建议、新测试用例编写方法与常见问题调试策略。

**更新** 测试基础设施已改进，使用全局常量 `__CALLKIT_TEST_MODE__` 替代 Vite define import.meta.env.*，提供更稳定的测试模式检测机制。测试环境现已更新至正式发布的1.0.0版本，确保与生产环境的一致性。switch-mode.mjs 脚本已增强，解决了 pnpm 安装跳过 file: 依赖的问题，通过强制清理缓存确保 tgz 包安装的可靠性。**新增** pnpm 锁文件依赖完整性哈希已更新，显著改善了测试环境依赖解析的稳定性，减少了因缓存问题导致的安装失败。

### 源码模式测试（test:source）
- 目标：直接使用 lib 源码作为依赖，避免打包产物差异带来的干扰，提升联调效率
- 关键点
  - 别名映射：将包名与样式文件指向 lib 目录，保证 import 与样式加载一致
  - 全局常量：通过 define 注入 `__CALLKIT_TEST_MODE__ = 'source'`，便于在演示应用中识别当前模式
  - 依赖来源：package.json 中删除对 .tgz 的依赖，确保解析到本地源码
- 典型流程
  - 运行 pnpm run test:source → 进入 test → 执行 switch:source → 执行 dev:source → 启动 Vite 源码模式配置

### switch-mode.mjs 切换机制
- 功能
  - 校验参数：仅允许 source 或 tgz
  - 源码模式：若存在 .tgz 依赖则删除，确保使用本地 lib
  - tgz 模式：校验 .tgz 是否存在，不存在则提示先执行 build:pack；存在则添加 file: 依赖
  - **增强** 缓存清理：强制清理 pnpm 缓存中的旧包，确保安装最新 tgz 文件
  - 输出提示：打印当前模式与后续操作建议
  - 版本固定：使用固定的1.0.0版本文件名，确保与生产环境一致
- 注意事项
  - 切换后需执行 pnpm install 或 pnpm add 以更新依赖树
  - tgz 模式依赖文件路径需与实际产物一致，使用release目录中的正式版本
  - **新增** 缓存清理机制确保 file: 依赖被正确安装
  - **新增** 依赖完整性哈希更新，减少缓存问题导致的安装失败

## 故障排查指南
- 切换失败
  - 症状：执行 switch:source 或 switch:tgz 后仍无法识别模式
  - 排查：确认已执行 pnpm install 或 pnpm add；检查 test/package.json 中依赖是否正确更新
- tgz 依赖缺失
  - 症状：tgz 模式提示找不到 .tgz 文件
  - 排查：先执行根项目 build:pack 生成 .tgz；核对文件路径与名称是否为 release/easemob-chat-callkit-vue3-1.0.0.tgz
- **新增** 缓存安装问题
  - 症状：pnpm 安装跳过 file: 依赖，导致 tgz 包未正确安装
  - 排查：switch-mode.mjs 已自动清理 pnpm 缓存，如果仍有问题，手动删除 node_modules 和 pnpm-store 后重新安装
  - **新增** 检查 pnpm 锁文件完整性哈希是否正确更新
- 样式或导出示错
  - 症状：样式未生效或组件导入失败
  - 排查：源码模式检查路径别名；tgz 模式检查 node_modules 解析
- 演示应用异常
  - 症状：登录失败、通话无法发起
  - 排查：检查环信 SDK 初始化与凭证；确认 Provider 配置与 debug 日志
- 版本不匹配
  - 症状：tgz 模式下版本号不正确
  - 排查：确认使用的是 release/easemob-chat-callkit-vue3-1.0.0.tgz 文件，而非开发版本
- **新增** 依赖解析问题
  - 症状：依赖安装缓慢或失败
  - 排查：检查 pnpm 锁文件完整性哈希；清理缓存后重试安装
  - 确认 pnpm 版本与锁文件格式兼容
